Palet

May 2015

iPhone/iPad (Swift)

Early in the summer I started developing in interest in mobile applications and UI design and so I thought I'd pick up some HTML/CSS/Javsscript. Likewise, I was creating graphics in Adobe Illustrator and using some of the same colors from Illustrator for web designs when I noticed how annoying it was to keep flipping through multiple applications and copying color hex-codes. I saw a mobile application that would serve as a small third workspace window as a way of increasing productivity when doing such tasks and so I developed Palet!, a mobile application with the ability to save colors and customize as many color schemes as the user decided. Palet was the very first mobile application that I developed and forced me to learn Swift and a little Objective-C and introduced me to some of the processes involved in creating mobile applications.

Mine Escape

June 2015

iPhone/iPad (Swift)

During the summer I was brainstorming ideas for mobile applications when I came across the idea of creating a short-term and sensory-memory aid that would be easy to use by both children and adults. I thought the easiest way to implement this would be through a simple tap-game. Likewise, I saw some elements in MineSweeper that I thought I could apply to the game, particularly its clean interface and ability to easily scale levels in size by simply increasing the dimensions of the game grid. However, I found that MineSweeper was more of a thinking game than a memory game and was a bit too slow-paced for a 9 year old to have any interest playing. This forced me to think about the types of games I used to enjoy playing when I was 9. One of those games was 'Match', where you are rewarded for remembering the locations of certain images. I developed MineEscape! to combine the sensory-memory of match with the basic interface and difficulty-scalability of MineSweeper.

Sphaudio

December 2015

iPhone/iPad, Web (Objective-C)

My latest application, Sphaudio (Spherical Audio) renders spheres in a three-dimensional space according to waveform data to provide an asthetically pleasing audio visualizer. The application uses the EZAudio framework to gather frequency and waveform data from the device's audio files which is then mapped to spherical objects which animate accordig to the data. The application uses the iOS device's library as the audio data source and thus the user is free to visualize any of the songs in his or her itunes music library. Likewise, the application also includes customization options.
